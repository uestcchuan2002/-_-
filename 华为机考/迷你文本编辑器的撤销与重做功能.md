# P5117.第2题-迷你文本编辑器的撤销与重做功能
## 时间限制
1000ms
## 题目内容
设计一个迷你文本编辑器，支持以下4种操作：
1. `APPEND x`：在文本末尾插入字符串`x`（`x`为无空格、无换行的连续字符）。
2. `POP`：删除文本末尾最后一个字符；若文本为空，该操作无效果，不存入撤销/重做历史。
3. `UNDO`：撤销上一次`APPEND`或`POP`操作，文本恢复至该操作执行前状态；可连续多次撤销，直到无操作可撤销。
4. `REDO`：重做上一次被撤销的操作，文本恢复至撤销前状态；可连续多次重做，直到无操作可重做。
    - 若执行`UNDO`后，新执行`APPEND`/`POP`，则清空全部重做历史，无法再执行重做。

初始文本为空。按顺序执行所有操作，输出最终文本内容。

## 输入描述
1. 第一行：正整数 $N$，操作总数量。
2. 后续 $N$ 行，每行一条操作：
    - `APPEND x`：末尾插入字符串`x`，`x`由大小写字母、数字、符号组成，不含空格换行；
    - `POP`：删除末尾单个字符；
    - `UNDO`：撤销操作；
    - `REDO`：重做操作。

### 输入约束
1. $1 \le N \le 200000$；
2. 单次插入字符串长度≤100，所有插入字符总数之和≤200000；
3. 操作序列合法：`UNDO`不会超过可撤销次数，`REDO`不会超过可重做次数。

## 输出描述
输出执行完所有操作后的完整文本；若文本为空，输出`nothing`。

## 样例1
### 输入
```
5
APPEND abc
APPEND d
UNDO
POP
REDO
```
### 输出
```
ab
```
### 说明
状态流转：`abc` → `abcd` → `abc` → `ab` → `ab`
撤销后执行新`POP`，重做历史被清空，REDO无效果。

---
## 样例2
### 输入
```
5
APPEND ab
UNDO
REDO
UNDO
REDO
```
### 输出
```
ab
```
### 说明
状态流转：`ab` → 空 → `ab` → 空 → `ab`
未插入新操作，撤销/重做可交替执行。
